# StockX Backend Guide

This project now has a fully working backend in `backend/` that powers login, live market feed, stock search, candle data, and paper trading persistence.

## Stack

- Node.js + Express API server
- CORS + JSON middleware
- File-based persistence (`backend/data/db.json`)
- Finnhub integration for real candles (with simulated fallback)

## Folder Structure

- `backend/src/server.js`: main API server and all REST routes
- `backend/src/store.js`: persistent state storage for users and sessions
- `backend/src/market.js`: live market simulation + candle fallback generator
- `backend/src/data.js`: stock master data and defaults

## High-Level Flow

1. Frontend logs in using `POST /api/auth/login`.
2. Backend creates a session token and returns it.
3. Frontend stores token in `localStorage` and sends it in `Authorization: Bearer ...`.
4. Backend authenticates protected routes through auth middleware.
5. Frontend pulls user state (`cashBalance`, `holdings`, `trades`, `watchlist`, `history`) from `GET /api/user/state`.
6. Live prices are fetched from `GET /api/market/live` every 3 seconds.
7. Trade and user actions are sent to backend routes, and backend writes updated state to `db.json`.

## Auth Theory

- This backend uses session-token auth (simple server-side session map).
- On login, a token is generated and mapped to user email.
- Protected routes validate token and load that user's state.
- This is enough for a local/dev app. For production, replace with secure password hashing + JWT/secure sessions + DB.

## Market Data Theory

- `GET /api/market/live` returns a snapshot of all symbols with:
  - current `price`
  - `prevClose`
  - `change`
  - `open/high/low`
  - `volume`
- Prices are generated by controlled random drift per symbol and held in memory so movements look continuous.

## Candle Data Theory

- Frontend calls `GET /api/stocks/:symbol/candles?period=...`.
- Backend attempts Finnhub first using:
  - header `x-api-key` sent from saved user key, or
  - `FINNHUB_API_KEY` from backend `.env`.
- If Finnhub fails or returns no candles, backend generates stable simulated candles so charts always render.

## Paper Trading Theory

- Buy route: validates qty/price/cash, updates holdings average cost, deducts cash, appends trade log.
- Sell route: validates available shares, credits cash, reduces/removes holdings, appends trade log.
- All updates are persisted, so data stays after restart.

## Search, Watchlist, History Theory

- `/api/stocks?query=...` provides backend-driven ticker/company search.
- `/api/user/watchlist/toggle` updates watchlist in persistent state.
- `/api/user/search-history` stores latest search symbols (max 10, de-duplicated).

## API Endpoints

### Public

- `GET /api/health`
- `POST /api/auth/login`
- `GET /api/stocks`
- `GET /api/market/live`
- `GET /api/stocks/:symbol/candles`

### Protected (Bearer token required)

- `GET /api/user/state`
- `POST /api/user/api-key`
- `POST /api/user/search-history`
- `POST /api/user/watchlist/toggle`
- `POST /api/trades/buy`
- `POST /api/trades/sell`

## Run Instructions

1. Install frontend dependencies:
   - `cd StockX`
   - `npm install`
2. Install backend dependencies:
   - `cd backend`
   - `npm install`
3. Start backend:
   - `npm run dev`
4. In another terminal, start frontend:
   - `cd StockX`
   - `npm run dev`

Frontend calls backend at `http://localhost:4000/api` by default.  
To change this, set `VITE_API_BASE_URL` in frontend environment.
